# KV 存储

> Nitro 提供了一个内置存储层，可以抽象文件系统、数据库或任何其他数据源。

::warning
Nitro v3 Alpha 文档仍在完善中 — 可能会有更新、不完善之处和偶尔的不准确。
::

Nitro 内置集成了 [unstorage](https://unstorage.unjs.io)，以提供与运行时无关的持久层。

## 用法

要使用存储层，可以使用 `useStorage()` 实用工具访问存储实例。

```ts
import { useStorage } from "nitro/storage";

// 默认存储（内存中）
await useStorage().setItem("test:foo", { hello: "world" });
const value = await useStorage().getItem("test:foo");

// 你可以在 useStorage(base) 中指定基路径
const testStorage = useStorage("test");
await testStorage.setItem("foo", { hello: "world" });
await testStorage.getItem("foo"); // { hello: "world" }

// 你可以使用泛型来定义返回值类型
await useStorage<{ hello: string }>("test").getItem("foo");
await useStorage("test").getItem<{ hello: string }>("foo");
```

:read-more{to="https://unstorage.unjs.io"}

### 可用方法

`useStorage()` 返回的存储实例提供以下方法：

| 方法 | 说明 |
|---|---|
| `getItem(key)` | 获取指定键的值。如果键不存在，返回 `null`。 |
| `getItems(items)` | 一次获取多个项。接受键数组或 `{ key, options }` 对象数组。 |
| `getItemRaw(key)` | 获取键的原始值（未解析）。适用于二进制数据。 |
| `setItem(key, value)` | 设置指定键的值。 |
| `setItems(items)` | 一次设置多个项。接受 `{ key, value }` 对象数组。 |
| `setItemRaw(key, value)` | 设置键的原始值（不序列化）。 |
| `hasItem(key)` | 检查键是否存在。返回布尔值。 |
| `removeItem(key)` | 从存储中移除指定键。 |
| `getKeys(base?)` | 获取所有键，可选按基路径过滤。 |
| `clear(base?)` | 清除所有键，可选按基路径过滤。 |
| `getMeta(key)` | 获取键的元数据（如 `mtime`, `atime`, `ttl`）。 |
| `setMeta(key, meta)` | 设置键的元数据。 |
| `removeMeta(key)` | 移除键的元数据。 |
| `mount(base, driver)` | 动态挂载存储驱动到基路径。 |
| `unmount(base)` | 卸载基路径上的存储驱动。 |
| `watch(callback)` | 监听变化。回调接受 `(event, key)`，事件为 `"update"` 或 `"remove"`。 |
| `unwatch()` | 停止监听变化。 |

也可用简写别名：`get`, `set`, `has`, `del`, `remove`, `keys`。

```ts
import { useStorage } from "nitro/storage";

// 获取某前缀下所有键
const keys = await useStorage("test").getKeys();

// 检查键是否存在
const exists = await useStorage().hasItem("test:foo");

// 移除键
await useStorage().removeItem("test:foo");

// 获取原始二进制数据
const raw = await useStorage().getItemRaw("assets/server:image.png");

// 获取元数据（类型，etag，修改时间等）
const meta = await useStorage("assets/server").getMeta("file.txt");
```

## 配置

你可以使用 `storage` 配置挂载一个或多个自定义存储驱动程序。  
键是挂载点名称，值是驱动程序名称和配置。

```ts [nitro.config.ts]
import { defineNitroConfig } from "nitro/config";

export default defineNitroConfig({
  storage: {
    redis: {
      driver: "redis",
      /* redis 连接器选项 */
    }
  }
})
```

然后，你可以使用 `useStorage("redis")` 函数访问 redis 存储。

你可以在 [unstorage 文档](https://unstorage.unjs.io/) 中找到驱动程序列表及其配置。

### 开发环境存储

你可以使用 `devStorage` 选项在开发和预渲染时覆写存储配置。

当生产环境的驱动不可用时（例如托管的 Redis 实例），这非常有用。

```ts [nitro.config.ts]
import { defineNitroConfig } from "nitro/config";

export default defineNitroConfig({
  storage: {
    db: {
      driver: "redis",
      host: "prod.example.com",
    }
  },
  devStorage: {
    db: {
      driver: "fs",
      base: "./.data/db"
    }
  }
})
```

在开发模式下，`devStorage` 的挂载将合并覆盖 `storage` 的挂载，使你可以使用本地文件系统驱动或内存驱动进行开发。

## 内置挂载点

Nitro 会自动挂载以下存储路径：

### `/assets`

服务器资源挂载在 `/assets` 基路径下。此挂载点提供对打包的服务器资源的只读访问（参见[服务器资源](#服务器资源)）。

```ts
import { useStorage } from "nitro/storage";

// 通过 /assets 挂载访问服务器资源
const content = await useStorage("assets/server").getItem("my-file.txt");
```

### 默认（内存中）

根存储（无基路径）默认使用内存驱动。此处存储的数据不会跨重启持久化。

```ts
import { useStorage } from "nitro/storage";

// 默认内存存储，不持久化
await useStorage().setItem("counter", 1);
```

要持久化数据，请使用 `storage` 配置挂载具有持久后端（如 `fs`, `redis` 等）的驱动。

## 服务器资源

Nitro 可将项目根目录下的 `assets/` 目录中的文件打包。这些文件可以通过 `assets/server` 存储挂载在运行时访问。

```
my-project/
  assets/
    data.json
    templates/
      welcome.html
  server/
    routes/
      index.ts
```

```ts [server/routes/index.ts]
import { useStorage } from "nitro/storage";

export default defineHandler(async () => {
  const serverAssets = useStorage("assets/server");

  const keys = await serverAssets.getKeys();
  const data = await serverAssets.getItem("data.json");
  const template = await serverAssets.getItem("templates/welcome.html");

  return { keys, data, template };
});
```

### 自定义资源目录

你可以使用 `serverAssets` 配置选项注册额外的资源目录：

```ts [nitro.config.ts]
import { defineNitroConfig } from "nitro/config";

export default defineNitroConfig({
  serverAssets: [
    {
      baseName: "templates",
      dir: "./templates",
    }
  ]
})
```

自定义资源目录可通过 `assets/<baseName>` 访问：

```ts
import { useStorage } from "nitro/storage";

const templates = useStorage("assets/templates");
const keys = await templates.getKeys();
const html = await templates.getItem("email.html");
```

### 资源元数据

服务器资源包含内容类型、ETag 和修改时间等元数据：

```ts
import { useStorage } from "nitro/storage";

const serverAssets = useStorage("assets/server");

const meta = await serverAssets.getMeta("image.png");
// { type: "image/png", etag: "\"...\"", mtime: "2024-01-01T00:00:00.000Z" }

// 便于设置响应头
const raw = await serverAssets.getItemRaw("image.png");
```

::note
开发环境中，服务器资源直接从文件系统读取。生产环境中，资源被打包并内联到构建输出。
::

## 运行时配置

默认情况下，Nitro 将在开发时使用文件系统驱动程序挂载项目目录和一些其他目录。

```js
// 访问项目根目录
const rootStorage = useStorage('root')

// 访问项目 src 目录（默认与根目录相同）
const srcStorage = useStorage('src')

// 访问服务器缓存目录
const cacheStorage = useStorage('cache')

// 访问临时构建目录
const buildStorage = useStorage('build')
```

> [!TIP]
> 你还可以使用 `devStorage` 键在开发期间覆盖存储配置。当你在生产中使用数据库并希望在开发中使用文件系统时，这非常有用。

为了使用 `devStorage` 键，你需要使用 `nitro dev` 命令，并且 `storage` 选项中的键必须与生产的相同。

```ts [nitro.config.ts]
import { defineNitroConfig } from "nitro/config";

export default defineNitroConfig({
  // 生产
  storage: {
    default: {
      driver: 'redis',
      /* redis 连接器选项 */
    }
  },
  // 开发
  devStorage: {
    default: {
      driver: 'fs',
      base: './data/kv'
    }
  }
})
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  nitro: {
    // 生产
    storage: {
      db: {
        driver: 'redis',
        /* redis 连接器选项 */
      }
    },
    // 开发
    devStorage: {
      db: {
        driver: 'fs',
        base: './data/db'
      }
    }
  }
})
```

## 运行时配置

在挂载点配置在运行时不确定的场景下，Nitro 可以在启动时动态添加挂载点，使用 [插件](/docs/plugins)。

```ts [plugins/storage.ts]
import { useStorage } from "nitro/storage";
import { definePlugin } from "nitro";
import redisDriver from "unstorage/drivers/redis";

export default definePlugin(() => {
  const storage = useStorage()

  // 动态传入运行时配置或其他来源的凭证
  const driver = redisDriver({
    base: "redis",
    host: process.env.REDIS_HOST,
    port: process.env.REDIS_PORT,
    /* 其他 redis 连接器选项 */
  })

  // 挂载驱动程序
  storage.mount("redis", driver)
})
```
