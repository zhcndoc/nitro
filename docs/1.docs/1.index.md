# 介绍

> Nitro 是一个全栈服务器框架，兼容任何运行时和任何部署目标。

Nitro 为你提供一个生产就绪的服务器，具备文件系统路由、代码拆分以及内置支持存储、缓存和数据库——全部运行时无关，并可部署到任何地方。

## 什么是 Nitro？

在 `routes/` 目录内创建服务器和 API 路由。每个文件直接映射到一个 URL 路径，Nitro 会处理剩下的工作——路由、代码拆分和优化构建。

你也可以通过创建一个 `server.ts` 文件完全掌控服务器入口。Nitro 的高层、运行时无关方法让你可以使用任何 HTTP 库，如 [Elysia](https://elysiajs.com/)、[h3](https://h3.dev) 或 [Hono](https://hono.dev)。

### 性能

Nitro 在构建时编译你的路由，消除了运行时路由器的需求。仅加载和执行处理每个入站请求所需的代码。这使其非常适合无服务器托管，无论项目大小，启动时间几乎为 0 毫秒。

### 随处部署

将你的服务器构建到兼容 Node.js、Bun、Deno 及众多托管平台的优化 `.output/` 文件夹，无需任何配置——包括 Cloudflare Workers、Netlify、Vercel 等。利用平台特性如 ESR、ISR 和 SWR，而无需修改一行代码。

### 服务器端渲染

使用你喜欢的模板引擎渲染 HTML，或直接在服务器上使用 React、Vue 或 Svelte 等组件库。实现完全的通用渲染并进行客户端水合。Nitro 提供基础和渐进式方法助你达成目标。

### 存储

Nitro 内置运行时无关的键值存储层。默认使用内存存储，但你可以连接 20 多种不同驱动（文件系统、Redis、S3 等），绑定到不同命名空间，并在不改代码的情况下切换它们。

### 缓存

Nitro 支持服务器路由和服务器函数的缓存，直接由服务器存储支持（通过 `cache` 命名空间）。

### 数据库

Nitro 还内置了 SQL 数据库。默认使用 SQLite，但你可以使用相同 API 连接并查询 10 多种数据库（Postgres、MySQL、PGLite 等）。

### 元框架基础

Nitro 可作为构建你自己的元框架的基础。Nuxt、SolidStart 和 TanStack Start 等流行框架全量或部分使用 Nitro。

## Vite 集成

Nitro 无缝集成 [Vite](https://vite.dev) 插件。如果你用 Vite 构建前端应用，加上 Nitro 你将获得 API 路由、服务器端渲染和完整的生产服务器——所有功能随着 `vite build` 一起构建。

```ts [vite.config.ts]
import { defineConfig } from "vite";
import { nitro } from "nitro/vite";

export default defineConfig({
  plugins: [nitro()],
});
```

使用 Nitro，`vite build` 会生成一个优化后的 `.output/` 文件夹，包含你的前端和后端——准备随时部署。
